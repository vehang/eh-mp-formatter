# 2026-04-04 Memory

## OpenClaw Docker Root 版本构建

### 背景
用户询问为什么 OpenClaw Docker 镜像使用 `node` 用户而不是 `root`，分析后认为作为智能体工具应该有高权限。

### 分析结论
- 官方镜像用 `node` 用户是基于安全考虑（容器逃逸防护）
- 但 OpenClaw 作为智能体需要安装软件、执行命令，权限限制会影响使用体验
- 建议提供选择权或默认 root 版本

### 执行的工作
1. 克隆项目: `https://github.com/justlovemaki/openclaw-docker-cn-im`
2. 修改 Dockerfile 创建纯 root 版本：
   - 删除所有 `USER node` 和 `USER root` 语句
   - 将所有 `/home/node` 路径改为 `/root`
   - 删除所有 `chown -R node:node` 操作
   - 删除 `gosu node` 降权逻辑
3. 修改 init.sh：
   - 移除 gosu 降权
   - 所有路径改为 /root
4. 添加国内镜像加速：
   - apt: `http://mirrors.aliyun.com`
   - npm: `https://registry.npmmirror.com`
   - pip: `https://pypi.tuna.tsinghua.edu.cn/simple`
   - Homebrew: `https://mirrors.tuna.tsinghua.edu.cn`

### 构建状态
- 镜像名称: `openclaw:2026.4.1-root` / `openclaw-root:latest`
- 构建目录: `~/.openclaw/workspace/dev/openclaw-docker-cn-im`
- 状态: 进行中 (npm 全局包安装阶段)

### 修改后的文件
- `~/.openclaw/workspace/dev/openclaw-docker-cn-im/Dockerfile`
- `~/.openclaw/workspace/dev/openclaw-docker-cn-im/init.sh`

### x86/ARM64 兼容性
- 基础镜像支持多架构
- apt/npm 包自动适配架构
- 无需额外修改即可在两种架构上构建
