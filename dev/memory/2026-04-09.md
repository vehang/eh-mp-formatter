# 2026-04-09 工作记录

## openclaw-easy 项目

### 版本管理
- 新增 `version.json`：包含 `versionName`（如 "1.0.0"）和 `versionCode`（如 1000）
- 版本检查接口：`GET https://api.yun.tilldream.com/api/nas/fw/getNewVersionV2?platform=openclaw-easy&versionCode=1000`

### API 接口
- `GET /api/version` - 获取当前版本信息
- `GET /api/update/check` - 检查更新
- `POST /api/update` - 一键更新（无参数，自动检查下载安装重启）
- `POST /api/fix` - 执行 `openclaw doctor --fix` 修复运行环境

### 自动更新任务
- 启动后延迟 30 秒检查更新
- 每 6 小时定时检查更新
- 环境变量控制：`AUTO_UPDATE`, `AUTO_UPDATE_ON_START`, `AUTO_UPDATE_INTERVAL`, `AUTO_UPDATE_INSTALL`

### 更新回滚机制
- `docker/update.sh`：独立更新脚本，不依赖 Node.js 进程
- `docker/start-check.sh`：启动检查脚本，检测失败更新并自动回滚
- 标记文件：`/tmp/openclaw-easy-update-marker`
- 更新失败时保留标记文件，下次启动自动回滚

### 构建目录统一
- 所有构建脚本统一到 `docker/` 目录
- `docker/build.sh` - 主构建脚本
- `docker/Dockerfile.integrated` - 整合版镜像

### 其他修改
- 移除保存配置时的自动重启逻辑
- `/api/config/simple` 和 `/api/weixin/bound` 使用异步重启
- 所有 API 返回格式统一为 `{ code, msg, currentTime, data }`
- SSE 事件格式统一

### 提交分支

- 主要提交：版本管理、自动更新、更新回滚机制
