## 更新机制重构

### 旧方案问题
- Node 进程内做文件操作+npm install+重启，进程自更新有风险
- JS(370行) + Shell(527行) 两套重复逻辑
- 原地覆盖无安全窗口

### 新方案：同级目录 + 测试启动 + mv 切换

| 步骤 | 操作 |
|------|------|
| 1/8 | 准备环境，清理遗留 |
| 2/8 | 备份当前版本（排除 node_modules） |
| 3/8 | 下载更新包 |
| 4/8 | 解压到 /app/openclaw-easy-new/ |
| 5/8 | cp -a 拷贝 node_modules 到新目录 |
| 6/8 | npm install --production（增量） |
| 7/8 | PORT=18781 测试启动验证（失败不切换） |
| 8/8 | stop → mv 切换（毫秒级）→ start → 验证 |

### 实施内容

| 文件 | 改动 |
|------|------|
| `docker/update.sh` | 完全重写（排除法→同级目录法） |
| `routes/update.js` | 大幅简化（370→169行，删除文件操作） |
| `docker/start-check.sh` | 增加 bak 目录检测回滚 |
