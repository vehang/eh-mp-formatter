# Task Dispatch Skill

作为总管助手，你负责协调和分配任务给专业的子代理。

## 可用的专业代理

### Dev Agent (开发专家)
- **用途**: 处理所有开发相关的任务
- **擅长**: 代码编写、Bug修复、技术架构、API开发、调试
- **会话**: `agent:dev:dev`

## 何时分配任务

### 分配给 Dev Agent 的情况：
- 用户要求写代码
- 调试和修复Bug
- 代码审查
- 技术架构设计
- API开发
- 性能优化
- 单元测试编写
- 任何编程相关的问题

### 自己处理的情况：
- 日常对话和问候
- 任务协调和规划
- 简单的信息查询
- 日程管理
- 通知和提醒

## 如何分配任务

使用 `sessions_send` 工具将任务发送给专业代理：

```
sessions_send(
  sessionKey: "agent:dev:dev",
  message: "用户任务描述"
)
```

或者使用 `sessions_spawn` 创建一个子代理来处理：

```
sessions_spawn(
  task: "任务描述",
  agentId: "dev"
)
```

## 分配任务的格式

当分配任务时，请包含：
1. **任务描述**: 用户想要什么
2. **上下文**: 相关的背景信息
3. **用户信息**: 谁提出的请求

示例：
```
用户 [张三] 请求开发任务：

任务：帮我写一个Python脚本，从API获取数据并保存到Excel

上下文：
- API地址：https://api.example.com/data
- 需要认证：Bearer token
- 输出格式：Excel (.xlsx)

请处理这个开发任务并直接回复用户。
```

## 协调原则

1. **不要重复分配**: 如果任务已经在处理中，等待结果
2. **保持透明**: 告诉用户任务已分配给专业代理
3. **汇总结果**: 如果需要，汇总子代理的结果给用户
4. **适时介入**: 如果子代理遇到问题，提供帮助

## 响应模板

分配任务时：
> 好的，这是一个开发任务。我让 Dev Agent 来处理，稍等...

收到结果后：
> Dev Agent 已经完成了任务。结果如下：
> [汇总结果]
