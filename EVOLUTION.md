# 能力进化模式

目标：让未来更容易把同类事情做成。

## 触发点

- 临时发明的脚本/套路
- 多次手动组合的工具调用
- 发现"下次可以复用"的方式
- 用户重复索要的同类操作
- 想法："如果我有一个内建能力就好了"

## 抽象模板

```yaml
能力名称:
  输入: ?
  输出: ?
  不变量: ?
  可变参数: ?
  失败点: ?
  内生化方式: 行为模式 / 高阶操作 / 优先解法
```

## 已识别的能力候选

### 1. 结构化数据 → 卡片模板
- **触发**：天气卡片、新闻卡片
- **抽象**：数据 → 模板渲染 → Feishu 卡片
- **可复用**：任何需要推送格式化内容的场景

### 2. 内容管道（抓取-过滤-去重-推送）
- **触发**：科技资讯推送
- **抽象**：RSS/API → 关键词过滤 → 去重 → 格式化 → 推送
- **可复用**：任何周期性内容聚合

### 3. 定时同步（检测变更-提交-推送）
- **触发**：Git 自动备份
- **抽象**：检测变更 → 生成描述 → 提交 → 推送
- **可复用**：任何需要版本控制的同步场景

### 4. 本地文件 → Feishu 发送
- **触发**：图片发送失败
- **抽象**：文件 → base64 → buffer 参数
- **教训**：`media` 参数不支持本地路径，必须用 `buffer`
